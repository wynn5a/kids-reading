# Single source of truth for lesson text & TTS audio

**Date:** 2026-07-03
**Status:** Approved

## Problem

The rendered lesson text and the pre-generated TTS clips can drift apart:

1. **Stale MP3s** — `pnpm gen:tts` skips a clip whenever its MP3 file exists,
   without checking whether the *text* changed. After a text change (e.g. a
   re-extraction or an override tweak) the page updates but the old audio keeps
   playing unless the file is deleted or everything is `--force`d.
2. **Duplicated `lineText`** — the tokens→sentence join exists twice: in
   `src/data/line-text.ts` and copied inline in `scripts/gen-tts.mjs`. If one
   changes, spoken text and rendered text silently diverge.
3. **No single command** — keeping page and audio in sync means remembering to
   run the extraction and the TTS generation separately.

## Decision: the PDF is the only truth

Text **and** pinyin both come from the textbook PDF. The chain is:

```
pdf/义务教育教科书·语文一年级上册.pdf
        │  scripts/extract_lessons.py   (decodes the book's own pinyin,
        ▼                                incl. 儿化: 会→huìr, 儿→"")
src/data/lessons.json                   (generated — never hand-edited)
        ├── page render: RubyText renders the tokens
        └── TTS: lineText(tokens) → MP3 per line (+ tts-overrides.json
            for spoken-only rewording of 多音字)
```

`lessons.json` is a build artifact. Adjusting text means adjusting the
extraction (or, for pronunciation only, `tts-overrides.json`) — never editing
`lessons.json` or audio files by hand.

Considered and rejected: a hand-editable `lessons.txt` source with
auto-generated pinyin (pinyin-pro). Rejected because the textbook's printed
pinyin (incl. 轻声, 儿化, deliberate no-pinyin passages in lesson 14) is
authoritative and must not be regenerated by a library.

## Changes

### 1. Content-aware TTS regeneration

`audio-manifest.json` already records the text each clip was generated from.
`gen-tts` now skips a clip only when **the MP3 exists AND the previous
manifest's text equals the current text** (unless `--force`). Editing one
line's text — via re-extraction or a `tts-overrides.json` change — regenerates
exactly that clip.

The skip/synthesize decision moves into a pure function in
`scripts/tts-plan.mjs`:

```
planClips(lessons, overrides, prevManifest, fileExists)
  → [{ key, text, action: "synth" | "skip" }]
```

unit-tested via Vitest (include pattern widened to cover
`scripts/**/*.test.mjs`).

### 2. One `lineText`

`scripts/gen-tts.mjs` imports `lineText` from `src/data/line-text.ts`
directly (Node ≥22.18 strips TS types natively; verified on v22.22). The
inline copy is deleted. One definition serves the app, its tests, and the
generation script.

### 3. API key only when needed

`gen-tts` requires `MIMO_API_KEY` only when at least one clip actually needs
synthesis, so a no-op sync check runs anywhere.

### 4. One command

- `pnpm gen:lessons` — runs the PDF extraction (`.pdfvenv/bin/python
  scripts/extract_lessons.py`)
- `pnpm gen` — `gen:lessons` then `gen:tts`; the full PDF→page+audio sync

### 5. Docs

CLAUDE.md and `scripts/README.md` describe the chain and the rule that
`lessons.json` is generated, not edited.

## Testing

- Unit tests for `planClips`: missing file → synth; existing file + same text
  → skip; existing file + changed text → synth; override change → synth;
  force → synth all; empty line → excluded.
- Existing suite (`pnpm test`) and `pnpm build` stay green.
- End-to-end: `pnpm gen:tts` on the unchanged repo reports all clips skipped
  and makes no API calls.
